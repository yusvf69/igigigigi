## 1. EXECUTIVE OVERVIEW

### 1.1 Purpose

Progress Tracking monitors every learner's journey through Islamic knowledge, measuring not just activity but genuine understanding and retention. It provides actionable insights to learners, parents, educators, and the platform itself to optimize the learning experience.

### 1.2 Core Principle

**Track understanding, not just activity. Measure what matters: retention, comprehension, and spiritual growth.**

---

## 2. TRACKING PHILOSOPHY

### 2.1 What We Track

| Category | Tracked | Not Tracked |
|---|---|---|
| **Knowledge** | Facts learned, concepts understood, mastery level | Time spent (secondary metric) |
| **Engagement** | Sessions, activities, consistency | Clicks, scrolling |
| **Performance** | Quiz accuracy, recall rate, speed | Comparison to others (private) |
| **Growth** | Knowledge expansion, difficulty increase | Mistakes (only as learning data) |
| **Spiritual** | Du'a tracking, Quran engagement (optional) | Personal worship details |

### 2.2 Privacy Principles

| Principle | Implementation |
|---|---|
| **Learner-Owned Data** | All progress data belongs to the learner |
| **Granular Sharing** | Learners control what is shared and with whom |
| **Anonymized Analytics** | Aggregate data for platform improvement |
| **No Selling** | Progress data never sold or monetized |
| **Right to Delete** | Complete data deletion available |

## 3. DATA MODEL

### 3.1 Core Entities

| Entity | Description | Key Fields |
|---|---|---|
| **Learner** | Platform user | id, level, joined_date, preferences |
| **KnowledgeFact** | Fact from Knowledge Graph | id, domain, difficulty, relationships |
| **MasteryRecord** | Learner's mastery of a fact | learner_id, fact_id, mastery_level, last_reviewed |
| **LearningEvent** | Single learning action | id, learner_id, event_type, timestamp, metadata |
| **Session** | Continuous learning period | id, learner_id, start, end, events[] |
| **Achievement** | Earned badge/challenge | id, learner_id, type, earned_date |
| **Goal** | Learner-set objective | id, learner_id, target, deadline, progress |

### 3.2 Mastery Levels

| Level | Label | Criteria | Color |
|---|---|---|---|
| 0 | Not Started | Fact never encountered | Gray |
| 1 | Exposed | Seen the fact once | Light blue |
| 2 | Learning | In active study phase | Blue |
| 3 | Familiar | Can recall with effort | Yellow |
| 4 | Mastered | Consistent correct recall | Green |
| 5 | Teaching | Can explain to others | Gold |

## 5. KNOWLEDGE MASTERY MODEL

### 5.1 Mastery Calculation

| Factor | Weight | Source |
|---|---|---|
| **Quiz Accuracy** | 0.35 | Last 5 quiz attempts on this fact |
| **Flashcard Recall** | 0.25 | Spaced repetition ease factor |
| **Recency** | 0.15 | Time since last correct recall |
| **Depth of Interaction** | 0.15 | Story/AI Teacher engagement with fact |
| **Consistency** | 0.10 | Pattern of correct responses over time |

### 5.2 Mastery Decay

| Time Since Last Review | Decay Factor |
|---|---|
| < 1 day | 1.0 (no decay) |
| 1-7 days | 0.95 |
| 7-30 days | 0.85 |
| 30-90 days | 0.70 |
| 90-365 days | 0.50 |
| > 365 days | 0.30 |

## 7. ANALYTICS PIPELINE

```
Raw Events (Kafka)
    │
    ▼
┌──────────────────────────────┐
│ Event Processor              │
│ • Validate                   │
│ • Enrich (add learner info)  │
│ • Route to appropriate store │
└──────────────────────────────┘
    │
    ├──► PostgreSQL (Mastery Records)
    ├──► Redis (Session state, cache)
    ├──► TimescaleDB (Time-series events)
    └──► Elasticsearch (Search, analytics)
                │
                ▼
┌──────────────────────────────┐
│ Analytics Engine             │
│ • Batch processing (hourly)  │
│ • Real-time aggregations     │
│ • ML-based predictions       │
└──────────────────────────────┘
    │
    ▼
┌──────────────────────────────┐
│ API Layer                    │
│ • Learner progress queries   │
│ • Dashboard data             │
│ • Report generation          │
└──────────────────────────────┘
```

## 9. PRIVACY AND DATA GOVERNANCE

| Policy | Implementation |
|---|---|
| **Data Ownership** | Learner owns all progress data |
| **Retention** | Data kept for account lifetime, deleted on request |
| **Anonymization** | Analytics data anonymized after 90 days |
| **Sharing Controls** | Granular per-data-type sharing permissions |
| **Export** | Full data export available in JSON/CSV |
| **Children's Privacy** | COPPA/GDPR-K compliance for under-13 |
