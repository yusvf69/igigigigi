## 1. EXECUTIVE OVERVIEW

### 1.1 Purpose

The Retrieval-Augmented Generation (RAG) pipeline is the critical bridge between the verified Knowledge Graph and the LLM-powered content generation and AI Teacher systems. It ensures that every piece of content generated by the platform is grounded in verified facts, preventing hallucination and maintaining the platform's commitment to factual accuracy.

### 1.2 Core Principle

**Every generated word must be traceable to a verified fact in the Knowledge Graph.**

The RAG pipeline enforces this principle by:
1. Retrieving relevant facts from the Knowledge Graph before generation.
2. Providing those facts as the exclusive context for LLM generation.
3. Verifying that generated content does not deviate from the provided facts.

### 1.3 RAG Pipeline Overview

```
    Query/Request
         |
    [Query Understanding]
         |
    [Query Expansion]
         |
    [Hybrid Retrieval]
      /          \
    [Vector]    [Keyword]
    [Search]    [Search]    [Graph Traversal]
      \          /            |
    [Fusion & Rerank]         |
         |                   |
    [Context Assembly] <------+
         |
    [LLM Generation]
         |
    [Grounding Check]
         |
    [Output]
```

## 2. RAG PHILOSOPHY

### 2.1 Design Principles

| Principle | Description |
|---|---|
| **Knowledge-First** | All retrieval starts from the Knowledge Graph. No web search or unverified sources. |
| **Precision over Recall** | It is better to retrieve fewer but highly relevant facts than many irrelevant ones. |
| **Provenance-Preserving** | Every retrieved fact retains its source citation and confidence score. |
| **Confidence-Gated** | Only facts above minimum confidence threshold are retrievable for generation. |
| **Multi-Strategy** | Different retrieval strategies for different use cases (semantic, keyword, graph). |
| **Verifiable Output** | Every generated output can be traced back to the specific facts used. |

### 2.2 When RAG is Used

| Use Case | RAG Type | Criticality |
|---|---|---|
| Content Generation (Story, Quiz, etc.) | Fact retrieval + context assembly | High |
| AI Teacher Conversation | Real-time retrieval | High |
| Fact Verification | Cross-reference retrieval | Critical |
| Question Answering | Direct retrieval | High |
| Translation | Terminology retrieval | Medium |
| Recommendation | Graph-based retrieval | Medium |

## 4. RETRIEVAL STRATEGIES

### 4.1 Retrieval Strategy Catalog

| Strategy | Method | Best For | Latency | Recall |
|---|---|---|---|---|
| **Vector Search** | Embedding similarity (cosine) | Semantic understanding | 20ms | High |
| **Keyword Search** | BM25 (Elasticsearch) | Exact term matching | 10ms | Medium |
| **Graph Traversal** | Cypher query | Relationship exploration | 50ms | Low |
| **Hybrid (Weighted)** | Vector 0.4 + Keyword 0.3 + Graph 0.3 | General purpose | 50ms | Highest |
| **Graph-First** | Start with graph, expand to vector/keyword | Verification, contradiction | 100ms | High |
| **Iterative** | Retrieve, generate, identify gaps, retrieve more | Complex questions | 200ms | Highest |
| **Structured** | Retrieve only specific fact types (rulings, definitions) | Quiz generation | 30ms | High |

### 4.2 Vector Search Strategy

```python
class VectorSearchStrategy:
    """
    Semantic search using embeddings in pgvector.
    """
    def __init__(self, embedding_model="text-embedding-3-small"):
        self.model = embedding_model
        self.index_type = "hnsw"  # or "ivfflat"
        self.distance_metric = "cosine"
    
    async def search(self, query: str, top_k: int = 20, 
                     filters: dict = None) -> list[SearchResult]:
        # Generate query embedding
        query_embedding = await self.embed(query)
        
        # Build SQL query with pgvector
        sql = """
            SELECT f.id, f.statement, f.confidence, f.domain,
                   1 - (e.embedding <=> $1) AS similarity
            FROM fact_embeddings e
            JOIN facts f ON f.id = e.fact_id
            WHERE f.confidence >= $2
              AND f.status = 'verified'
            """
        
        # Add filters (domain, fact_type, etc.)
        if filters:
            sql += self._build_filter_clause(filters)
        
        sql += " ORDER BY similarity DESC LIMIT $3"
        
        results = await self.db.query(sql, query_embedding, 
                                      self.min_confidence, top_k)
        return [SearchResult(**r) for r in results]
```

### 4.3 Graph Traversal Strategy

```cypher
// Example: Retrieve context around a concept
MATCH (c:Concept {name: $concept_name})
// Get facts directly related
OPTIONAL MATCH (c)-[r]-(f:Fact)
WHERE f.confidence >= $min_confidence
// Get scholarly positions
OPTIONAL MATCH (c)<-[:about]-(sp:ScholarlyPosition)
// Get supporting source citations
OPTIONAL MATCH (f)-[:cites]->(s:Source)
// Return context
RETURN {
  concept: c,
  facts: collect(DISTINCT {fact: f, relationship: type(r), source: s}),
  scholarly_positions: collect(DISTINCT sp)
} AS context
LIMIT 1
```

## 7. CONTEXT ASSEMBLY

### 7.1 Context Assembly Strategy

```python
class ContextAssembler:
    """
    Assembles retrieved facts into a structured context for LLM generation.
    """
    
    def __init__(self, max_tokens: int = 8000):
        self.max_tokens = max_tokens
        self.system_prompt = self._load_system_prompt()
    
    async def assemble(self, retrieved_facts: list[Fact], 
                       query: str, task_type: str) -> Context:
        
        # 1. Sort facts by relevance and confidence
        facts = sorted(retrieved_facts, 
                      key=lambda f: (f.relevance * 0.7 + f.confidence * 0.3),
                      reverse=True)
        
        # 2. Group facts by theme/domain if multiple domains
        grouped = self._group_by_theme(facts)
        
        # 3. Format facts with provenance
        formatted_groups = []
        total_tokens = 0
        
        for theme, theme_facts in grouped.items():
            group_text = f"## {theme}\n"
            group_tokens = self._count_tokens(group_text)
            
            for fact in theme_facts:
                fact_text = self._format_fact(fact)
                fact_tokens = self._count_tokens(fact_text)
                
                if total_tokens + group_tokens + fact_tokens > self.max_tokens:
                    break
                
                group_text += fact_text
                group_tokens += fact_tokens
            
            formatted_groups.append(group_text)
            total_tokens += group_tokens
        
        # 4. Assemble final context
        context = Context(
            system_prompt=self.system_prompt,
            retrieved_facts=formatted_groups,
            total_facts_used=len(facts),
            confidence_range=(min(f.confidence for f in facts),
                            max(f.confidence for f in facts)),
            token_budget_used=total_tokens,
            token_budget_total=self.max_tokens
        )
        
        return context
    
    def _format_fact(self, fact: Fact) -> str:
        return (
            f"- **{fact.statement}** "
            f"[Confidence: {fact.confidence:.0%}] "
            f"[Source: {fact.source_title}, "
            f"{fact.source_author}] "
            f"[{fact.source_reference}]\n"
        )
```

## 8. GROUNDING AND VERIFICATION

### 8.1 Grounding Check Algorithm

```python
class GroundingChecker:
    """
    Verify that generated content is grounded in retrieved facts.
    """
    
    async def check(self, generated_text: str, 
                    retrieved_facts: list[Fact]) -> GroundingReport:
        # 1. Extract atomic claims from generated text
        claims = await self._extract_claims(generated_text)
        
        # 2. For each claim, find matching fact(s)
        ungrounded_claims = []
        grounded_claims = []
        
        for claim in claims:
            best_match = await self._find_matching_fact(claim, retrieved_facts)
            
            if best_match and best_match.similarity >= self.threshold:
                grounded_claims.append(GroundedClaim(
                    claim=claim,
                    matched_fact=best_match.fact,
                    similarity=best_match.similarity
                ))
            else:
                ungrounded_claims.append(UngroundedClaim(
                    claim=claim,
                    reason="No matching fact found in retrieved context"
                ))
        
        # 3. Calculate grounding score
        grounding_score = len(grounded_claims) / len(claims) if claims else 1.0
        
        return GroundingReport(
            total_claims=len(claims),
            grounded_claims=grounded_claims,
            ungrounded_claims=ungrounded_claims,
            grounding_score=grounding_score,
            passed=grounding_score >= self.min_grounding_score
        )
    
    async def _extract_claims(self, text: str) -> list[str]:
        """Use LLM to extract atomic factual claims from generated text."""
        prompt = f"""
        Extract all atomic factual claims from the following text. 
        A claim is a single factual assertion that can be verified.
        Return as a JSON array of strings.
        
        Text: {text}
        """
        response = await self.llm.extract(prompt)
        return json.loads(response)
    
    async def _find_matching_fact(self, claim: str, 
                                  facts: list[Fact]) -> Match:
        """Find the best matching fact for a claim."""
        # Generate embedding for claim
        claim_embedding = await self.embed(claim)
        
        best_match = None
        for fact in facts:
            similarity = cosine_similarity(claim_embedding, fact.embedding)
            if best_match is None or similarity > best_match.similarity:
                best_match = Match(fact=fact, similarity=similarity)
        
        return best_match
```

## 13. HYBRID SEARCH

### 13.1 Fusion Algorithm

```python
class HybridFusion:
    """
    Combine results from multiple retrieval strategies.
    """
    
    def fuse(self, vector_results: list, keyword_results: list, 
             graph_results: list, weights: tuple = (0.4, 0.3, 0.3)) -> list:
        
        # Reciprocal Rank Fusion (RRF)
        k = 60  # RRF constant
        
        scores = {}
        
        for rank, result in enumerate(vector_results):
            scores[result.id] = scores.get(result.id, 0) + weights[0] * (1 / (rank + k))
        
        for rank, result in enumerate(keyword_results):
            scores[result.id] = scores.get(result.id, 0) + weights[1] * (1 / (rank + k))
        
        for rank, result in enumerate(graph_results):
            scores[result.id] = scores.get(result.id, 0) + weights[2] * (1 / (rank + k))
        
        # Sort by fused score
        ranked = sorted(scores.items(), key=lambda x: x[1], reverse=True)
        
        return ranked[:self.top_k]
```

## 14. RERANKING

### 14.1 Reranking Pipeline

```
    Initial Results (50 facts)
         |
    [Cross-Encoder Scoring]
    - Pairwise (query, fact) relevance scoring
    - Model: Cohere rerank-v3 or custom cross-encoder
         |
    [Diversity Selection (MMR)]
    - Maximum Marginal Relevance
    - Balance relevance with novelty
    - Avoid duplicate/similar facts
         |
    [Confidence Filtering]
    - Remove facts below confidence threshold
    - Boost facts with human scholarly approval
         |
    [Domain Filtering]
    - Ensure coverage of required domains
    - Remove out-of-scope domains
         |
    [Token Budget Truncation]
    - Keep top facts up to context window limit
         |
    Final Context (10-20 facts)
```

### 14.2 MMR Algorithm

```python
def mmr_rerank(documents: list, query_embedding: list, 
               lambda_param: float = 0.7, top_k: int = 15) -> list:
    """
    Maximum Marginal Relevance for diversity.
    lambda_param: 1.0 = pure relevance, 0.0 = pure diversity
    """
    selected = []
    candidate_pool = documents.copy()
    
    while len(selected) < top_k and candidate_pool:
        scores = []
        for doc in candidate_pool:
            relevance = cosine_similarity(query_embedding, doc.embedding)
            
            if selected:
                diversity = min(1 - cosine_similarity(doc.embedding, s.embedding) 
                              for s in selected)
            else:
                diversity = 1.0
            
            mmr_score = lambda_param * relevance - (1 - lambda_param) * (1 - diversity)
            scores.append(mmr_score)
        
        best_idx = scores.index(max(scores))
        selected.append(candidate_pool.pop(best_idx))
    
    return selected
```

    FOR SELECT
    USING (confidence >= 0.85 AND status = 'verified');
