## 1. INSTRUCTIONAL DESIGN MODELS

### 1.1 ADDIE Model

The ADDIE model (Analysis, Design, Development, Implementation, Evaluation) is the foundational instructional design framework. It provides a systematic, iterative approach to creating effective learning experiences.

```
┌─────────────────────────────────────────────────────────────────────┐
│                      ADDIE PROCESS MODEL                             │
│                                                                      │
│   ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│   │ ANALYSIS │───►│  DESIGN  │───►│DEVELOPMENT│───►│IMPLEMENT │───►│EVALUATION│
│   │          │    │          │    │          │    │          │    │          │
│   │ • Needs  │    │ • Obj.   │    │ • Create  │    │ • Deliver│    │ • Form.  │
│   │ • Learner│    │ • Assess │    │ • Media   │    │ • Facil. │    │ • Summ.  │
│   │ • Context│    │ • Strat. │    │ • Author  │    │ • Tech   │    │ • Confirm│
│   │ • Task   │    │ • Seq.   │    │ • Quality │    │ • Support│    │ • Improve│
│   └──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
│         │              │               │               │              │       │
│         └──────────────┴───────────────┴───────────────┴──────────────┘       │
│                              │ Feedback Loop                                  │
│                              ▼                                                 │
│                    Continuous Improvement                                     │
└─────────────────────────────────────────────────────────────────────────────┘
```

#### 1.1.1 When to Use ADDIE

| Context | Suitability | Rationale |
|---|---|---|
| Large-scale curriculum development | High | Structured, documented, quality-controlled |
| Regulatory/compliance training | High | Traceable, auditable process |
| Content with high accuracy requirements | High | Multiple quality gates |
| Islamic knowledge content | Very High | Verification and scholarly review require structure |
| Rapid development needed | Low | Linear approach can be slow |
| Highly innovative content | Medium | Structure can constrain creativity |
| Small, simple modules | Low | Overhead exceeds benefit |

Cross-Reference: See LXP-CURR-001 (Curriculum Paths) for ADDIE-based curriculum development workflow.

### 1.2 SAM (Successive Approximation Model)

SAM is an agile alternative to ADDIE, emphasizing iterative prototyping and frequent feedback cycles.

```
┌─────────────────────────────────────────────────────────────────────┐
│                   SAM PROCESS MODEL                                   │
│                                                                      │
│   ┌──────────┐                                                       │
│   │ PREPARE  │                                                       │
│   │ PHASE    │                                                       │
│   │ • Gather │                                                       │
│   │   info   │                                                       │
│   │ • Initial│                                                       │
│   │   design │                                                       │
│   └────┬─────┘                                                       │
│        │                                                             │
│        ▼                                                             │
│   ┌──────────────────────────────────────────────────────────────┐  │
│   │                  ITERATIVE DESIGN PHASE                        │  │
│   │                                                               │  │
│   │   ┌──────────┐    ┌──────────┐    ┌──────────┐              │  │
│   │   │ SAVVY    │───►│ PROTOTYPE│───►│ REVIEW   │              │  │
│   │   │ START    │    │          │    │          │              │  │
│   │   │ • Design │    │ • Create │    │ • Review │              │  │
│   │   │   start  │    │ • Story  │    │ • Feedback│             │  │
│   │   └──────────┘    └──────────┘    └──────────┘              │  │
│   │                        │               │                     │  │
│   │                        └───────┬───────┘                     │  │
│   │                                │                             │  │
│   │                           (Repeat)                            │  │
│   └──────────────────────────────────────────────────────────────┘  │
│                                │                                    │
│                                ▼                                    │
│   ┌──────────────────────────────────────────────────────────────┐  │
│   │               ITERATIVE DEVELOPMENT PHASE                     │  │
│   │                                                               │  │
│   │   • Extend prototype to full content                          │  │
│   │   • Continuous review and revision                            │  │
│   │   • Alpha → Beta → Gold                                      │  │
│   └──────────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────────┘
```

#### 1.2.1 When to Use SAM

| Context | Suitability | Rationale |
|---|---|---|
| Rapid content development | High | Quick iterations, fast delivery |
| Innovative or experimental content | High | Feedback-driven refinement |
| Learner preferences are uncertain | High | Test with prototypes before full build |
| Complex stakeholder needs | High | Continuous alignment through reviews |
| Content requiring scholarly validation | Medium | Can be efficient but needs structured review points |
| Large-scale consistent curriculum | Low | Risk of inconsistency across iterations |

### 1.3 Backward Design (Understanding by Design)

Backward Design (Wiggins & McTighe, 2005) starts with desired outcomes and works backward:

```
┌─────────────────────────────────────────────────────────────────────┐
│                    BACKWARD DESIGN PROCESS                           │
│                                                                      │
│   STAGE 1                    STAGE 2                    STAGE 3     │
│   Identify                    Determine                  Plan        │
│   Desired                     Acceptable                 Learning    │
│   Results                     Evidence                   Experiences │
│                                                                      │
│   ┌──────────────────┐      ┌──────────────────┐      ┌─────────┐  │
│   │ • Goals/Standards│      │ • Performance    │      │ • Teach- │  │
│   │ • Understandings │      │   Tasks          │      │   ing   │  │
│   │ • Essential      │─────►│ • Criteria       │─────►│ • Learn-│  │
│   │   Questions      │      │ • Assessments    │      │   ing   │  │
│   │ • Knowledge/Skills│     │ • Self-Evaluation│      │ • Activ-│  │
│   └──────────────────┘      └──────────────────┘      │   ities │  │
│                                                        └─────────┘  │
│                                                                      │
│   Key Question: What do we want learners to ultimately be able to    │
│   do with this knowledge?                                            │
└─────────────────────────────────────────────────────────────────────┘
```

#### 1.3.1 When to Use Backward Design

| Context | Suitability | Rationale |
|---|---|---|
| Outcome-focused curricula | Very High | Prioritizes transferable understanding |
| Islamic value-based education | Very High | Focus on application (amal) not just knowledge |
| Performance assessment design | High | Clear alignment to outcomes |
| Unit and course design | High | Good for macro-level design |
| Individual lesson design | Medium | May be too abstract for micro-design |

#### 1.3.2 Islamic Education Application

| Stage | Islamic Learning Focus | Example |
|---|---|---|
| Stage 1: Desired Results | What should a Muslim know, understand, and do? | "Learners will understand that tawheed is the foundation of all Islamic worship and practice" |
| Stage 2: Evidence | How will learners demonstrate understanding? | "Learners will analyze a real-world situation and identify whether it aligns with tawheed" |
| Stage 3: Plan | What learning experiences lead to outcomes? | "Stories of prophets teaching tawheed, followed by discussion of contemporary applications" |

### 1.4 Dick & Carey Model

The Dick & Carey model is a comprehensive, systems-oriented approach with ten interconnected components:

| Component | Description | Islamic Education Application |
|---|---|---|
| 1. Instructional Goals | Broad outcomes from needs assessment | "Learners will be able to perform salah correctly according to authentic fiqh" |
| 2. Instructional Analysis | Identify subordinate skills and knowledge | Breaking down salah into prerequisites: wudu, conditions, pillars |
| 3. Entry Behaviors | Learner characteristics and prior knowledge | Current salah knowledge, age, language proficiency |
| 4. Performance Objectives | Specific, measurable objectives | "Given a checklist, demonstrate all 12 conditions of salah with 100% accuracy" |
| 5. Criterion-Referenced Items | Test items aligned to objectives | Checklist-based performance assessment |
| 6. Instructional Strategy | Sequence and structure of instruction | Presentation → practice → feedback → transfer |
| 7. Instructional Materials | Content and media | Videos, diagrams, text, interactive practice |
| 8. Formative Evaluation | Testing during development | Expert review, one-to-one, small group testing |
| 9. Summative Evaluation | Overall effectiveness | Post-instruction assessment, learner satisfaction |
| 10. Revision | Data-driven refinement | Update content based on evaluation findings |

#### 1.4.1 When to Use Dick & Carey

| Context | Suitability | Rationale |
|---|---|---|
| Systematic curriculum development | Very High | Complete, interconnected framework |
| Content with clear skill hierarchies | Very High | Hierarchical analysis is central |
| High-stakes training programs | High | Rigorous evaluation component |
| Islamic knowledge with clear prerequisites | Very High | Many Islamic subjects have clear hierarchies |
| Rapid development | Low | Detailed analysis can be time-consuming |

### 1.5 Rapid Prototyping

Rapid prototyping builds quick, rough versions of instruction to test with learners before full development.

| Phase | Activity | Output |
|---|---|---|
| Needs Assessment | Quick identification of core needs | Problem statement |
| Prototype | Build a small, functional prototype | Working sample (e.g., 1 story, 5 questions) |
| Learner Testing | Test with 3-5 target learners | User feedback, performance data |
| Refinement | Revise based on feedback | Improved prototype |
| Full Development | Expand to full content | Complete instructional unit |
| Evaluation | Assess effectiveness | Evaluation report |

#### 1.5.1 When to Use Rapid Prototyping

| Context | Suitability | Rationale |
|---|---|---|
| Novel instructional approaches | High | Test before committing resources |
| Learner preferences are uncertain | High | Get real user feedback early |
| Time and budget constraints | High | Reduces risk of building wrong solution |
| Islamic content with multiple audiences | Medium | Different cultural contexts may need different prototypes |
| Content requiring scholarly approval | Medium | Prototypes can help scholars visualize but final approval needed |

### 1.6 Model Selection Framework

The IEP uses a hybrid approach, selecting and combining models based on project characteristics:

```
┌─────────────────────────────────────────────────────────────────────┐
│               MODEL SELECTION DECISION MATRIX                        │
│                                                                      │
│  Project Characteristic          Recommended Model                   │
│  ──────────────────────          ──────────────────                  │
│  Large curriculum build          ADDIE + Backward Design             │
│  Rapid feature release           SAM + Rapid Prototyping             │
│  New content type                SAM + Rapid Prototyping             │
│  High-accuracy requirements      ADDIE + Dick & Carey                │
│  Learner-centered redesign       Backward Design + SAM               │
│  Complex skill hierarchy         Dick & Carey + ADDIE                │
│  Islamic scholarly content       ADDIE + Backward Design             │
│  Mobile-first experience         SAM + Rapid Prototyping             │
│  Compliance/verification needs   ADDIE + Dick & Carey                │
│  Experimental/narrative content  SAM + Backward Design               │
│                                                                      │
│  Default Model for IEP: ADDIE with SAM-style iterations              │
│  (Rigorous structure of ADDIE + Agile flexibility of SAM)           │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.7 Islamic Education Context Considerations

| Consideration | Design Implication | Model Adaptation |
|---|---|---|
| Source Authenticity | Verification stages needed in process | ADDIE with extended Evaluation phase for scholarly review |
| Traditional Methods | Blend modern and traditional approaches | Backward Design with traditional outcomes |
| Spiritual Dimension | Include tazkiyah and spiritual outcomes | Backward Design Stage 1 includes spiritual outcomes |
| Multiple Scholarly Positions | Design for presenting ikhtilaf | SAM prototyping to test presentation approaches |
| Language Complexity | Arabic integration planning | Analysis phase must include language analysis |
| Cultural Sensitivity | Cultural review gates in process | ADDIE with cultural review at Design and Development |

## 2. ADDIE PROCESS

### 2.1 Analysis Phase

The Analysis phase answers: "What is needed, for whom, and why?"

#### 2.1.1 Needs Assessment

| Analysis Type | Question | Method | Output |
|---|---|---|---|
| Performance Gap | What is the gap between current and desired performance? | Stakeholder interviews, learner surveys, assessment analysis | Needs assessment report |
| Instructional vs Non-Instructional | Is instruction the right solution? | Root cause analysis | Solution recommendation |
| Feasibility | Can we develop this instruction? | Resource assessment, timeline, budget | Feasibility report |
| Priority | What needs are most critical? | Impact vs effort analysis | Prioritized needs list |

#### 2.1.2 Learner Analysis

| Dimension | Questions | Data Sources |
|---|---|---|
| Demographics | Age, language, location, education level | User profiles, registration data |
| Prior Knowledge | What do learners already know? | Diagnostic assessments, learner model |
| Learning Preferences | Preferred modalities, study times, pace | Behavioral analytics, surveys |
| Motivation | Why are they learning? Goals? | Goal setting data, engagement metrics |
| Cultural Background | Cultural context of learner's Islam | Profile data, language preferences |
| Accessibility Needs | Any accommodations needed? | Accessibility settings, user input |
| Technology Access | Device, internet, platform familiarity | System analytics |

#### 2.1.3 Context Analysis

| Factor | Questions | Analysis Method |
|---|---|---|
| Physical Environment | Where will learning occur? | Learner surveys, usage analytics |
| Technical Environment | What devices, OS, bandwidth? | Platform telemetry |
| Social Context | Is learning individual or group? | Social feature usage data |
| Time Constraints | How much time available? | Session duration analytics |
| Support Resources | Is help available? | Support system data |

#### 2.1.4 Task Analysis

| Analysis Type | Focus | Method |
|---|---|---|
| Topic Analysis | What are the key concepts? | Subject matter expert consultation |
| Procedural Analysis | What steps are involved? | Observation, documentation review |
| Conceptual Analysis | What are the critical relationships? | Concept mapping with experts |
| Information Processing Analysis | What mental processes are required? | Think-aloud protocols |

### 2.2 Design Phase

The Design phase answers: "How will we achieve the learning outcomes?"

#### 2.2.1 Design Components

| Component | Description | Output |
|---|---|---|
| Learning Objectives | Specific, measurable outcomes | Objective statement for each lesson |
| Assessment Strategy | How will learning be measured? | Assessment blueprint |
| Instructional Strategy | What approach will be used? | Strategy plan |
| Content Sequencing | Order of content delivery | Scope and sequence document |
| Media Selection | What formats and channels? | Media specification |
| Lesson Structure | How will individual lessons be organized? | Lesson template |
| Feedback Design | How will learners receive feedback? | Feedback specification |

#### 2.2.2 Design Documentation

```
DESIGN DOCUMENT — Lesson/Module Template

Title: [Module/Lesson Name]
Target Audience: [Age, Level, Prerequisites]
Duration: [Estimated time to complete]

LEARNING OBJECTIVES:
  1. [Objective 1 — Bloom's level: ___ ]

  Sequence:
    1. Engagement/Gain Attention
    2. Activate Prior Knowledge
    3. Present New Content

    5. Independent Practice
    6. Application/Transfer
    7. Assessment

  - Primary Modality: [Story, Quiz, Interactive, etc.]
  - Supporting Media: [Audio, Video, Diagrams, etc.]
  - Accessibility Requirements:

  - Knowledge Graph Facts: [KG-FACT-XXXXX]

REVIEW REQUIREMENTS:
  - Subject Matter Expert: [Yes/No]
  - Scholar Review: [Yes/No]
  - Learner Testing: [Yes/No]
```

| Visual Content | AI illustration + design review | Islamic art guidelines, accuracy, appeal |

| Animation | Storyboard → Animation → Review | Accuracy, pacing, clarity |

### 2.4 Implementation Phase

The Implementation phase answers: "How do we deliver and facilitate the instruction?"

#### 2.4.1 Implementation Components

| Component | Description | Considerations |
|---|---|---|
| Delivery Platform | Technical deployment | Mobile app, web, or both; offline capability |
| User Onboarding | Introduction to the system | First-time experience, orientation, initial assessment |
| Facilitation | How is instruction supported? | AI Mentor guidance, human teacher support |
| Schedule | When does learning happen? | Self-paced, scheduled, blended |
| Monitoring | How is progress tracked? | Real-time dashboards, intervention triggers |
| Support | What support is available? | Technical support, pedagogical support, community |

#### 2.4.2 Delivery Quality Requirements

| Requirement | Standard | Verification Method |
|---|---|---|
| Platform uptime | 99.9% | System monitoring |
| Page load time | < 3 seconds | Performance testing |
| Media playback | < 2 second start time | Streaming tests |
| Offline availability | All core content | Offline testing |
| Accessibility | WCAG 2.1 AA compliance | Accessibility audit |
| Cross-device compatibility | iOS, Android, Web | Device testing matrix |

### 2.5 Evaluation Phase

The Evaluation phase answers: "Did the instruction work?"

#### 2.5.1 Evaluation Types

| Evaluation Type | When | What It Measures | Method |
|---|---|---|---|
| Formative | During development | Quality, usability, effectiveness | Expert review, learner testing |
| Summative | After implementation | Learning outcomes achievement | Pre/post testing, assessments |
| Confirmative | After extended use | Long-term retention, transfer | Follow-up assessments (30, 90, 180 days) |
| Impact | After program completion | Behavioral change, real-world application | Surveys, interviews, observational data |
| ROI | After program completion | Cost vs. benefit analysis | Cost tracking, outcome valuation |

#### 2.5.2 Formative Evaluation Points

| Stage | Evaluation Method | Participants | Criteria |
|---|---|---|---|
| Design Review | Expert review of design document | Instructional designers, SMEs | Alignment, completeness, feasibility |
| Alpha Review | SME review of content | Subject matter experts | Accuracy, completeness |
| Scholar Review | Scholarly verification | Islamic scholars | Authenticity, orthodoxy, source accuracy |
| Beta Testing | Learner testing | Target audience learners | Usability, engagement, comprehension |
| Pilot | Full implementation with sample | Representative learner group | All metrics (learning, engagement, satisfaction) |

Cross-Reference: See LXP-ASSESS-001 (Assessment Science) for detailed evaluation design.

### 2.6 ADDIE for Islamic Content Development

## 3. LEARNING OBJECTIVES

### 3.1 Bloom's Taxonomy (Revised)

The revised Bloom's Taxonomy (Anderson & Krathwohl, 2001) organizes cognitive processes from simple to complex:

```
┌─────────────────────────────────────────────────────────────────────┐
│              REVISED BLOOM'S TAXONOMY                                │
│                                                                      │
│                    ┌─────────────────────┐                           │
│                    │     CREATE          │                           │
│                    │  Produce new work   │                           │
│                    └─────────────────────┘                           │
│                              ▲                                       │
│                    ┌─────────────────────┐                           │
│                    │     EVALUATE        │                           │
│                    │ Justify a decision  │                           │
│                    └─────────────────────┘                           │
│                              ▲                                       │
│                    ┌─────────────────────┐                           │
│                    │     ANALYZE         │                           │
│                    │ Draw connections    │                           │
│                    └─────────────────────┘                           │
│                              ▲                                       │
│                    ┌─────────────────────┐                           │
│                    │     APPLY           │                           │
│                    │ Use in new situation│                           │
│                    └─────────────────────┘                           │
│                              ▲                                       │
│                    ┌─────────────────────┐                           │
│                    │   UNDERSTAND        │                           │
│                    │ Explain ideas       │                           │
│                    └─────────────────────┘                           │
│                              ▲                                       │
│                    ┌─────────────────────┐                           │
│                    │     REMEMBER        │                           │
│                    │ Recall facts        │                           │
│                    └─────────────────────┘                           │
│                                                                      │
│              Knowledge Dimension:                                    │
│              • Factual Knowledge (terminology, details)               │
│              • Conceptual Knowledge (theories, models, categories)    │
│              • Procedural Knowledge (methods, skills, techniques)     │
│              • Metacognitive Knowledge (self-knowledge, strategies)   │
└─────────────────────────────────────────────────────────────────────┘
```

#### 3.1.1 Verbs by Level

| Level | Definition | Verb Examples | Islamic Education Examples |
|---|---|---|---|
| Remember | Retrieve relevant knowledge from long-term memory | List, define, identify, name, recall, state | "List the five pillars of Islam" |
| Understand | Construct meaning from instructional messages | Explain, describe, summarize, interpret, classify | "Explain the meaning of tawheed al-rububiyyah" |
| Apply | Carry out or use a procedure in a given situation | Implement, execute, demonstrate, use, solve | "Demonstrate the correct method of performing wudu" |
| Analyze | Break material into parts and detect relationships | Compare, contrast, differentiate, organize, attribute | "Compare the conditions of salah between different madhahib" |
| Evaluate | Make judgments based on criteria and standards | Evaluate, critique, assess, justify, prioritize | "Evaluate the strength of this hadith chain" |
| Create | Put elements together to form a coherent whole | Design, construct, develop, produce, compose | "Design a plan for teaching the concept of tawheed to children" |

### 3.2 SMART Objectives

| Criterion | Description | Weak Objective | Strong Objective |
|---|---|---|---|
| Specific | Clearly defined, unambiguous | "Learn about tawheed" | "Define the three categories of tawheed" |
| Measurable | Observable, quantifiable | "Understand salah" | "Demonstrate all 12 conditions of salah on a checklist" |
| Achievable | Realistic given constraints | "Become a scholar of fiqh in one month" | "Master the Fiqh of Salah module (score > 80%)" |
| Relevant | Connected to broader goals | "Learn Arabic grammar" | "Learn Arabic grammar rules needed to understand Surah al-Fatiha" |
| Time-bound | Specific timeframe | "Complete the Quran" | "Memorize Juz Amma (30th juz) within 6 months" |

### 3.3 Objective Classification System

Each learning objective in the IEP is classified along three dimensions:

| Dimension | Categories | Purpose |
|---|---|---|
| Cognitive Level | Remember → Understand → Apply → Analyze → Evaluate → Create | Determines depth of learning |
| Knowledge Type | Factual, Conceptual, Procedural, Metacognitive | Determines nature of content |
| Islamic Domain | Quran, Hadith, Fiqh, Aqidah, Seerah, Arabic, Akhlaq, Tazkiyah | Determines content source |

#### 3.3.1 Objective Classification Matrix

```
Classification: [Domain].[Cognitive Level].[Knowledge Type]

Example Classifications:
- Fiqh.Remember.Factual: "Recall the five pillars of Islam"
```

## 4. TAXONOMY DESIGN

### 4.1 Cognitive Domain for Islamic Education

The cognitive domain addresses knowledge and intellectual skills. For Islamic education, it must be adapted to include:

| Level | Islamic Adaptation | Example Objective |
|---|---|---|
| Remember | Memorization of Quran, hadith, duas is a distinct skill with special importance | "Memorize Surah al-Mulk with correct tajweed" |
| Understand | Deep comprehension of divine speech and prophetic tradition | "Explain the contextual meaning of this ayah" |
| Apply | Application of Islamic knowledge in daily life | "Perform salah correctly according to the sunnah" |
| Analyze | Critical analysis of sources, chains, and arguments | "Analyze the chain of narration for reliability" |
| Evaluate | Scholarly evaluation of positions and evidence | "Evaluate the evidence for different fiqh positions" |
| Create | Ijtihad, synthesis of multiple sources, teaching | "Develop a lesson plan teaching this concept to beginners" |

### 4.2 Affective Domain for Islamic Education

The affective domain (Krathwohl, 1964) addresses attitudes, values, and emotions. This is particularly important for Islamic education:

| Level | Description | Islamic Application | Example Outcome |
|---|---|---|---|
| Receiving | Awareness and willingness to attend | Being open to Islamic learning | "Learner willingly engages with Islamic content" |
| Responding | Active participation and reaction | Engagement with Islamic practices | "Learner voluntarily performs sunnah prayers" |
| Valuing | Commitment to a value or belief | Internalizing Islamic values | "Learner prioritizes acts of worship in daily life" |
| Organization | Integrating values into a system | Building coherent Islamic worldview | "Learner integrates Islamic values into life decisions" |
| Characterization | Value system controls behavior | Islamic character becomes identity | "Learner's behavior consistently reflects Islamic teachings" |

### 4.3 Psychomotor Domain for Islamic Education

The psychomotor domain addresses physical skills, with several Islamic applications:

| Level | Description | Islamic Application | Example |
|---|---|---|---|
| Imitation | Observing and copying | Learning prayer movements | "Copy the teacher's prayer positions" |
| Manipulation | Performing from instruction | Guided practice | "Perform wudu following step-by-step guidance" |
| Precision | Accurate, controlled performance | Correct, consistent practice | "Recite with accurate tajweed independently" |
| Articulation | Coordinated, smooth performance | Fluent worship | "Complete full salah sequence smoothly with khushu" |
| Naturalization | Automatic, effortless performance | Unconscious competence | "Recite Quran with perfect tajweed without conscious effort" |

### 4.4 Knowledge Dimension

| Knowledge Type | Description | Islamic Education Example |
|---|---|---|
| Factual Knowledge | Basic elements, terminology, details | "The five pillars are shahada, salah, zakat, sawm, hajj" |
| Conceptual Knowledge | Interrelationships among elements | "Understanding how tawheed relates to shirk and how shirk invalidates worship" |
| Procedural Knowledge | How to do something | "Steps for performing wudu, criteria for hadith classification" |
| Metacognitive Knowledge | Knowledge about cognition | "Knowing that one learns best through stories and reflection" |

## 5. CONTENT SEQUENCING

### 5.1 Simple-to-Complex Sequencing

Content progresses from simpler to more complex concepts, building cognitive structures incrementally.

| Sequence Type | Description | Islamic Example |
|---|---|---|
| Part-to-Whole | Teach components before whole system | Individual fiqh rulings → Fiqh of salah → Fiqh of ibadat |
| Simple-to-Elaborate | Basic concept before nuances | "Allah is One" → Types of tawheed → Subtle violations |
| Concrete-to-Abstract | Tangible examples first | Stories of prophets → Abstract aqidah principles |
| Known-to-Unknown | Known concepts as foundation | Compare Islamic monotheism to familiar concepts |
| Foundation-to-Specialized | Core before advanced | Core beliefs → Detailed theology |

### 5.2 Prerequisite Hierarchy Design

Every learning objective must have its prerequisites clearly defined:

```
EXAMPLE PREREQUISITE HIERARCHY: FIQH OF SALAH

Level 5: Advanced Salah Fiqh
  - Making up missed prayers
  - Combining prayers (jam')
│
Level 4: Full Salah
  - Full salah sequence
  - Conditions of salah (12 items)
  - Pillars of salah (14 items)
  - Sunnah actions of salah
│
Level 3: Salah Components
  - Positions of salah
  - Recitations in salah
  - Movements in salah
│
Level 2: Purification (Taharah)
  - Wudu (ablution)
  - Ghusl (full bath)
  - Tayammum (dry ablution)
  - Removing najasah (impurity)
│
Level 1: Foundations
  - What is salah?
  - Why do we pray?
  - When do we pray? (times)
  - Facing the qiblah
```

### 5.3 Spiral Curriculum Architecture

The spiral curriculum (Bruner, 1960) revisits concepts at increasing levels of complexity:

| Visit | Age/Level | Depth | Focus | Content Approach |
|---|---|---|---|---|
| 1st Visit | Age 6-8 | Surface | Awareness | Stories: "Allah is One" |
| 2nd Visit | Age 9-12 | Basic understanding | Comprehension | Simple definitions, examples |
| 3rd Visit | Age 13-15 | Detailed | Analysis | Categories, comparisons |
| 4th Visit | Age 16-18 | Advanced | Evaluation | Scholarly opinions, evidence |
| 5th Visit | Adult | Expert | Synthesis | Ijtihad, teaching, application |

#### 5.3.1 Spiral Curriculum Example: Tawheed

| Level | Content | Cognitive Level |
|---|---|---|
| Level 1 (Age 6-8) | "Allah is One — there is no god but Allah" | Remember |
| Level 2 (Age 9-12) | Three categories: Rububiyyah, Uluhiyyah, Asma wa Sifat | Understand |
| Level 3 (Age 13-15) | Evidence for each category from Quran and Sunnah | Analyze |
| Level 4 (Age 16-18) | Conflicts with tawheed: shirk, bidah, kufr | Evaluate |
| Level 5 (Adult) | Contemporary challenges to tawheed and responses | Evaluate + Create |

### 5.4 Scaffolding Progression

| Phase | Support Level | Learner Independence | Example |
|---|---|---|---|
| Modeling | Maximum | Minimal | "Watch how I classify this hadith" |
| Guided Practice | High | Low | "Now you try, I'll help when needed" |
| Collaborative Practice | Medium | Medium | "Work with a peer to classify these hadith" |
| Independent Practice | Low | High | "Classify these hadith on your own" |
| Application | Minimal | Maximum | "Find and classify new hadith from a collection" |

### 5.5 Just-in-Time Instruction

Just-in-time instruction provides information exactly when the learner needs it, rather than all upfront.

| Scenario | Just-in-Time Content | Traditional Alternative | Advantage |
|---|---|---|---|
| Learner encounters unfamiliar Arabic term in a story | Translation appears on hover or tap | Pre-teach all Arabic vocabulary | Reduces upfront load, connects to context |
| Learner fails a quiz question on a specific condition | Instant mini-lesson on that condition | Full lesson on all conditions before quiz | Targeted, efficient remediation |
| Learner asks AI Teacher a specific question | AI provides relevant knowledge from KG | Learner must search through content | Immediate, contextual response |

### 5.6 Sequencing for Islamic Knowledge

| Islamic Discipline | Optimal Sequence | Rationale |
|---|---|---|
| Quran | Juz Amma → Short surahs → Full juz → Entire Quran | Progressive length and difficulty |
| Tajweed | Letters → Rules → Application → Mastery | Skills build on each other |
| Fiqh | Ibadat → Muamalat → Contemporary | Worship first (practical), then transactions |
| Hadith | Arba'in (40 hadith) → Selected collections → Full Sahih | Core content first, then expanded |
| Aqidah | Tawheed → Risalah → Qadr → Ghayb | Foundation principles first |
| Seerah | Makkan → Madinan → Entire Seerah | Chronological is natural |
| Arabic | Alphabet → Words → Sentences → Grammar → Texts | Language acquisition sequence |

## 6. ASSESSMENT DESIGN

### 6.1 Formative vs Summative Assessment

| Characteristic | Formative Assessment | Summative Assessment |
|---|---|---|
| Purpose | Improve learning during instruction | Evaluate learning after instruction |
| Timing | During learning | After learning |
| Frequency | Continuous, frequent | Periodic, end of unit/module |
| Stakes | Low or no stakes | Higher stakes |
| Feedback | Immediate, detailed, specific | Delayed, summary |
| Learner Role | Active participant | Test taker |
| Teacher/AI Role | Coach, guide | Evaluator |
| Examples | Practice quizzes, discussions, reflections | End-of-unit test, comprehensive exam |

#### 6.1.1 Formative Assessment Types in IEP

| Type | Description | Islamic Application |
|---|---|---|
| Knowledge Check | Quick quiz after content presentation | 3-5 questions after a story or AI Teacher session |
| Self-Assessment | Learner rates own understanding | "How confident are you in explaining this concept?" |
| Peer Discussion | Collaborative knowledge check | "Explain this ruling to a partner and get feedback" |
| Teach-Back | Learner teaches AI or peer | "Explain the conditions of salah to the AI" |
| Practice Exercise | Low-stakes practice | Apply fiqh ruling to scenarios |
| Reflection Entry | Written reflection | "What was the most important thing you learned?" |

#### 6.1.2 Summative Assessment Types in IEP

| Type | Description | Islamic Application |
|---|---|---|
| End-of-Unit Quiz | Comprehensive unit assessment | 20-30 questions covering all unit objectives |
| Mastery Check | Demonstrate complete understanding | Score > 80% on topic assessment |
| Performance Assessment | Demonstrate skill | Perform wudu correctly on video submission |
| Portfolio | Collection of learning evidence | Collected reflections, assignments, quizzes |
| Capstone Project | Comprehensive application | Research paper, teaching demonstration |
| Ijazah Assessment | Traditional mastery verification | Complete oral examination with scholar |

### 6.2 Diagnostic vs Placement Assessment

| Type | Purpose | Timing | Example |
|---|---|---|---|
| Diagnostic | Identify knowledge gaps and misconceptions | Before new instruction | "Which of these topics do you already understand?" |
| Placement | Determine appropriate starting level | Before curriculum entry | "Take this assessment to determine your level" |
| Pre-Assessment | Baseline for growth measurement | Before instruction | "Score before and after to measure learning" |
| Readiness Check | Verify prerequisite knowledge | Before new module | "Do you know the conditions of salah before studying pillars?" |

#### 6.2.1 Diagnostic Assessment Design

```
DIAGNOSTIC ASSESSMENT PROTOCOL

Purpose: Identify existing knowledge, gaps, and misconceptions

Structure:
  Phase 1: Self-Report (2 min)
    "I can explain tawheed to someone else" [Yes/No/Not sure]
    "I know the three categories of tawheed" [Yes/No/Not sure]
  
  Phase 2: Knowledge Check (5 min)
    5-10 questions across the domain
    Mix of easy, medium, and hard items
  
  Phase 3: Misconception Detection (3 min)
    Purposefully designed items targeting common misconceptions
    "Which of the following is an example of shirk in worship?"
  
Output:
  - Mastery profile per topic area
  - Identified misconceptions
  - Recommended starting point
  - Suggested review content
```

### 6.3 Criterion vs Norm-Referenced

| Characteristic | Criterion-Referenced | Norm-Referenced |
|---|---|---|
| Standard | Fixed standard of achievement | Relative to peer group |
| Purpose | Has learner mastered the material? | How does learner compare? |
| Score Interpretation | Absolute (80% = mastery) | Relative (85th percentile) |
| Content Coverage | Specific, aligned to objectives | Broad, sampling of domain |
| Item Selection | Items represent objectives | Items that discriminate |
| Islamic Application | "Have you mastered this knowledge?" | "How does your progress compare?" |

#### 6.3.1 IEP Assessment Approach

The IEP primarily uses **criterion-referenced** assessment with optional norm-referenced features:

| Assessment Purpose | Primary Approach | Optional Features |
|---|---|---|
| Mastery verification | Criterion-referenced (score > 80%) | Compare to age group norms (optional) |
| Progress tracking | Criterion-referenced (growth over time) | Percentile rank (optional, private) |
| Placement | Criterion-referenced (meets prerequisites) | Norms for level determination |
| Gamification | Norm-referenced (leaderboards) | Optional, focus on personal improvement |

### 6.4 Authentic Assessment

Authentic assessment measures ability to apply knowledge in real-world contexts:

| Criterion | Traditional Assessment | Authentic Assessment |
|---|---|---|
| Task | Choose correct answer | Perform in realistic context |
| Context | Artificial testing environment | Real-world scenario |
| Process | Single correct answer | Multiple valid approaches |
| Outcome | Score or grade | Demonstration of competence |
| Feedback | Right/wrong | Performance feedback |

#### 6.4.1 Authentic Assessment Examples for Islamic Education

| Knowledge Area | Traditional Assessment | Authentic Assessment |
|---|---|---|
| Fiqh of Salah | MCQ: "What invalidates salah?" | Scenario: "You are in a situation where you cannot face the qiblah. What do you do?" |
| Zakat Calculation | "Calculate zakat on $10,000" | "Here is a family's financial portfolio for the year. What zakat is due?" |
| Seerah | "When did the Hijrah occur?" | "Imagine you are a companion in Makkah in 622 CE. What factors would you consider?" |
| Aqidah | "Define tawheed" | "Someone says 'Whatever Allah wills' but also consults fortune tellers. Analyze this." |

### 6.5 Performance Assessment

Performance assessment requires learners to demonstrate skills rather than select answers:

| Assessment Type | Description | Islamic Application | Evaluation Method |
|---|---|---|---|
| Demonstration | Perform a procedure | Perform wudu correctly | Checklist-based observation |
| Oral Examination | Verbal response to questions | Traditional shafahi (oral exam) | Rubric-based scholar evaluation |
| Product Creation | Create something | Design a lesson on tawheed | Rubric-based evaluation |
| Teaching | Instruct others | Teach a peer about salah | Observation + feedback |
| Case Analysis | Analyze a case | Analyze a contemporary fiqh issue | Written analysis rubric |

### 6.6 Assessment Blueprint

Every assessment in the IEP follows a blueprint:

```
ASSESSMENT BLUEPRINT

Assessment Title: [Name]
Type: [Formative/Summative/Diagnostic/Placement]
Domain: [Islamic discipline]
Target Level: [Bloom's level]
Duration: [Time estimate]

OBJECTIVE COVERAGE:
  Objective 1 (Remember) — 2 items (20%)
  Objective 2 (Understand) — 3 items (30%)
  Objective 3 (Apply) — 3 items (30%)
  Objective 4 (Analyze) — 2 items (20%)

ITEM DISTRIBUTION:
  Easy (recall): 3 items (30%)
  Medium (application): 4 items (40%)
  Hard (analysis/evaluation): 3 items (30%)

TOTAL ITEMS: 10
MASTERY THRESHOLD: 80%

ITEM FORMATS:
  MCQ: 5 items
  Matching: 2 items
  Scenario-based: 2 items
  Open-ended: 1 item

ADAPTIVE PARAMETERS:
  Starting difficulty: Medium
  Adaptation rule: Correct → harder; Incorrect → easier

```

## 7. FEEDBACK DESIGN

### 7.1 Immediate vs Delayed Feedback

| Characteristic | Immediate Feedback | Delayed Feedback |
|---|---|---|
| Timing | Right after response | After a delay (minutes to days) |
| Effect on Learning | Better for procedural skills | Better for long-term retention of declarative knowledge |
| Motivation | Maintains engagement, reduces frustration | Can increase retrieval effort |
| Best For | Practice, drill, skill acquisition | Deep understanding, transfer, complex tasks |
| Islamic Application | Tajweed correction, wudu practice | Fiqh reasoning, tafsir understanding |

#### 7.1.1 Feedback Timing Guidelines

| Learning Phase | Recommended Timing | Rationale |
|---|---|---|
| Initial acquisition | Immediate | Prevents error reinforcement |
| Skill development | Immediate to short delay | Corrects errors while still relevant |
| Fluency building | Short delay (seconds) | Promotes some retrieval effort |
| Application | Moderate delay (minutes to hours) | Encourages deeper processing |
| Transfer | Long delay (days) | Maximizes retrieval practice effect |
| Assessment | After full attempt | Allows complete cognitive processing |

### 7.2 Directive vs Facilitative Feedback

| Characteristic | Directive Feedback | Facilitative Feedback |
|---|---|---|
| Approach | Tells what to do | Guides discovery |
| Example | "You forgot to check the intention — you need niyyah" | "What condition of salah might relate to your internal state?" |
| Cognitive Load | Low (learner receives answer) | Higher (learner must process) |
| Learner Autonomy | Low | High |
| Best For | Novices, urgent correction | Advanced learners, conceptual understanding |
| Islamic Application | "The answer is tawheed al-rububiyyah" | "Think about what category of tawheed deals with Allah's actions" |

#### 7.2.1 Directive-Facilitative Continuum

```
DIRECTIVE ◄───────────────────────────────────────────► FACILITATIVE

Level 1: Explicit correction
  "The correct answer is [X]"

Level 2: Error identification + hint
  "You made an error in step 3. Check the intention condition."

Level 3: Error location only
  "Review step 3 again."

Level 4: General prompt
  "Check the conditions of salah — you may have missed one."

Level 5: Strategy prompt
  "Try going through each condition one by one."

Level 6: Metacognitive prompt
  "What strategy could help you verify all conditions?"

Level 7: Open question
  "What do you think you might have missed and why?"

Level 8: Self-discovery
  "Compare your answer to the standard — what differences do you notice?"
```

### 7.3 Verification vs Elaboration Feedback

| Characteristic | Verification Feedback | Elaboration Feedback |
|---|---|---|
| Content | Whether answer is correct | Why answer is correct/incorrect |
| Length | Brief | Extended |
| Learning Effect | Low (only confirms) | High (explains and teaches) |
| Example | "Correct!" or "Incorrect" | "Incorrect. The correct answer is A because the Prophet ﷺ said..." |

#### 7.3.1 Elaboration Types

| Elaboration Type | Description | Example |
|---|---|---|
| Correct Response | Provide correct answer | "The correct answer is tawheed al-uluhiyyah" |
| Rationale | Explain why | "Because uluhiyyah refers to the worship of Allah alone" |
| Source Evidence | Provide source | "As Allah says in Quran: 'And your God is One God' (2:163)" |
| Conceptual Link | Connect to other concepts | "This connects to tawheed al-rububiyyah because..." |
| Common Error | Explain common mistake | "Many learners confuse this with rububiyyah. The key difference is..." |
| Application | Show real-world relevance | "This means in daily life, we direct all worship to Allah alone" |

### 7.4 Error-Specific Feedback

| Error Type | Feedback Strategy | Example |
|---|---|---|
| Factual Error | Correct + Provide source | "That's not correct. The correct fact is... [source]" |
| Conceptual Misunderstanding | Explain concept + Contrast | "You're confusing X with Y. Here's how they differ..." |
| Procedural Error | Step-by-step correction | "The correct order is A → B → C. You did B before A." |
| Prerequisite Gap | Identify gap + Recommend review | "This question requires understanding of X. Let's review that first." |
| Careless Mistake | Recognize effort + Redirect | "You knew this — check your answer again." |
| Overgeneralization | Provide boundary conditions | "That's generally true, but here's when it doesn't apply..." |

### 7.5 Praise and Criticism Design

#### 7.5.1 Effective Praise vs Ineffective Praise

| Characteristic | Effective Praise | Ineffective Praise |
|---|---|---|
| Focus | Effort, strategy, improvement | Ability, outcome, ease |
| Specificity | Specific to the achievement | Generic ("Good job!") |
| Sincerity | Genuine and earned | Routine or excessive |
| Timing | After the achievement | Delayed or out of context |
| Attribution | Controllable factors | Fixed ability |

#### 7.5.2 Praise Examples

| Situation | Effective Praise | Ineffective Praise |
|---|---|---|
| Correct answer after effort | "Your persistence with this concept really paid off!" | "You're so smart!" |
| Improvement over time | "I can see your understanding growing — you've improved 15% this week!" | "You're getting better." |
| Using good strategy | "Breaking that hadith into chain and text was a great approach!" | "Great job!" |
| Helping others learn | "You explained that concept very clearly — teaching helps you learn deeper." | "You're a good teacher." |

#### 7.5.3 Constructive Correction

| Principle | Example | Avoid |
|---|---|---|
| Specific + Actionable | "The timing of the intention needs correction — it should be at the beginning, not during" | "That was wrong" |
| Effort-Recognizing | "You identified most conditions correctly. Let's look at the two you missed." | "You missed two conditions." |
| Improvement-Focused | "Next time, try checking each condition against the checklist before submitting." | "You need to be more careful." |
| Growth-Mindset | "This is a challenging concept. Let's try a different approach." | "Some people just don't get this." |

### 7.6 Feedback Timing and Frequency

| Feedback Context | Timing | Frequency | Type |
|---|---|---|---|
| Flashcard review | Immediate | Every response | Verification + brief elaboration |
| Quiz (practice) | Immediate | Every question | Verification + elaboration on incorrect |
| Quiz (summative) | After completion | End of quiz | Score + item-level review |
| Open-ended question | Within 24 hours | Per response | Elaboration + directive for improvement |
| Peer learning | Delayed (reviewed by AI) | Per interaction | Facilitative, metacognitive |
| Daily reflection | End of session | Once per session | Summary + process feedback |

### 8.2 Modality Principles

| Principle | Description | Application |
|---|---|---|
| Modality Effect | Words in audio better than text when paired with visuals | Narrate diagram labels, don't show as text |
| Redundancy Principle | Don't present identical audio and text | Either narrate or display text, not both |
| Split-Attention Principle | Integrate related information | Place labels on diagrams, not in separate legend |
| Segmenting Principle | Break into learner-paced segments | Short sections with continue button |
| Pre-Training Principle | Teach key terms and components first | Introduce key concepts before full instruction |
| Signaling Principle | Highlight key information | Use arrows, bold, color coding |

### 8.3 Multimedia Learning Theory

Multimedia learning (Mayer, 2001) applies cognitive theory to the design of multimedia:

```
┌─────────────────────────────────────────────────────────────────────┐
│                    MULTIMEDIA LEARNING                              │
│                                                                      │
│  Words     ───────────────────► Verbal Channel  ──────►              │
│                                                                      │
│  Pictures  ───────────────────► Visual Channel  ──────►              │
│                                                                      │
│  Three assumptions:                                                  │
│  1. Dual Channels: Visual and auditory are separate                  │
│  2. Limited Capacity: Each channel has limited capacity              │
│  3. Active Processing: Meaningful learning requires cognitive        │
│     processing (selecting, organizing, integrating)                  │
└─────────────────────────────────────────────────────────────────────┘
```

### 8.4 Cognitive Theory of Multimedia Learning

| Process | Description | Support Strategy |
|---|---|---|
| Selecting Words | Attending to relevant verbal information | Clear narration, minimal extraneous audio |
| Selecting Images | Attending to relevant visual information | Clear diagrams, highlight key elements |
| Organizing Words | Building verbal mental model | Logical narration sequence |
| Organizing Images | Building visual mental model | Clear visual structure, relationships |
| Integrating | Connecting verbal and visual models | Present simultaneously, reference each other |

### 8.5 Mayer's Principles of Multimedia Design

| Principle | Description | Application to Islamic Content |
|---|---|---|
| Coherence | Remove extraneous material | Avoid decorative graphics that don't teach |
| Signaling | Highlight essential material | Use callouts on key Islamic concepts |
| Redundancy | Avoid duplicate presentations | Either narrate or show text, not both |
| Spatial Contiguity | Place related text near graphics | Labels directly on diagrams |
| Temporal Contiguity | Present narration and graphics simultaneously | Animated diagrams with synchronized narration |
| Segmenting | Break presentation into segments | Short lesson segments with breaks |
| Pre-Training | Pre-teach key terms | Introduce Arabic terms before using them |
| Modality | Use narration for graphics, not text | Narrated animation rather than on-screen text |
| Multimedia | Use both words and pictures | Illustrated stories and animated guides |
| Personalization | Use conversational style | "You" and "your" in AI Teacher dialogue |
| Voice | Use human voice for narration | Warm, clear narration voice |
| Image | Don't add speaker image to screen | Avoid decorative speaker images |

| Content Type | Optimal Media Mix | Rationale |

| Quran Memorization | Audio + Text + Calligraphy | Auditory is primary; visual supports script recognition |

## 12. INSTRUCTIONAL STRATEGIES

### 12.1 Direct Instruction

Direct instruction is teacher-centered, systematic, and explicit.

| Phase | Activity | AI Teacher Role |
|---|---|---|
| Opening | State objectives, activate prior knowledge | "Today we will learn the conditions of salah. What do you already know?" |
| Presentation | Present new content clearly and sequentially | "The first condition of salah is..." (with visuals) |
| Guided Practice | Lead practice with prompts and corrections | "Now, let's check each condition together. Number 1 is..." |
| Independent Practice | Learner practices alone with feedback | "Your turn: list all conditions. I'll check." |
| Closure | Review key points, preview next | "Today we learned 5 conditions. Tomorrow we'll learn 3 more." |

### 12.2 Inquiry-Based Learning

Inquiry-based learning centers on learner questions and investigation.

| Phase | Activity | AI Teacher Role |
|---|---|---|
| Question | Present guiding question | "Why do Muslims fast during Ramadan?" |
| Investigation | Learner explores resources to answer | Guide to relevant resources; suggest search paths |
| Discovery | Learner constructs understanding | Confirm or redirect; ask further questions |
| Discussion | Share and compare findings | "How does your understanding compare to the scholarly position?" |
| Reflection | Reflect on process and learning | "What did you learn about how to investigate Islamic questions?" |

### 12.3 Problem-Based Learning

Problem-based learning presents authentic problems as the starting point.

| Phase | Activity | Islamic Example |
|---|---|---|
| Problem Presentation | Present real-world scenario | "Ahmed is traveling during Ramadan. He is unsure about fasting rules." |
| Problem Analysis | Identify what is known and unknown | What do we know about travel and fasting? What do we need to find? |
| Research | Gather relevant Islamic evidence | Search Quran, hadith, fiqh rulings on travel and fasting |
| Solution Development | Formulate response | Determine valid options for the traveler |
| Presentation | Share and defend solution | Present reasoning with evidence |
| Debrief | Compare with scholarly position | AI Teacher presents normative fiqh position |

### 12.4 Project-Based Learning

Project-based learning involves sustained investigation resulting in a tangible product.

| Element | Description | Islamic Education Example |
|---|---|---|
| Driving Question | Compelling problem to solve | "How can we share the story of the Prophet's mercy with our community?" |
| Sustained Inquiry | Long-term investigation | Research Seerah, collect examples of mercy, understand application |
| Authenticity | Real-world relevance | Create content for actual community use |
| Learner Choice | Meaningful decisions | Choose format (video, article, presentation), audience |
| Reflection | Metacognitive processing | What did I learn about the Prophet's character? About myself? |
| Critique & Revision | Iterative improvement | Peer and expert feedback; refine product |
| Public Product | Share with real audience | Present at community event, share online |

### 12.5 Case-Based Learning

Case-based learning presents detailed cases for analysis:

| Step | Description | Fiqh Example |
|---|---|---|
| Case Presentation | Detailed scenario | "A woman finds money. She asks scholars. One says keep, another says give charity..." |
| Individual Analysis | Learner analyzes independently | Identify principles, evidence, reasoning |
| Small Group Discussion | Share analysis with peers | Compare interpretations |
| Whole Class Discussion | Synthesize insights | AI Teacher guides synthesis |
| Debrief | Expert analysis | Scholar's analysis of the case |
| Reflection | Generalize principles | Extract principles applicable to similar cases |

### 12.6 Discovery Learning

Discovery learning allows learners to explore and construct understanding:

| Type | Description | Islamic Application |
|---|---|---|
| Inductive Discovery | Examples → Rules | Present 10 situations where salah is valid/invalid; learner discovers conditions |
| Deductive Discovery | Rules → Examples | Given conditions of salah, learner finds examples meeting/breaking them |
| Guided Discovery | Structured exploration | "Try to categorize these actions under conditions, pillars, or sunnah" |
| Exploratory Discovery | Free exploration | Explore Knowledge Graph connections between concepts |

```
┌─────────────────────────────────────────────────────────────────────┐
│                    KIRKPATRICK EVALUATION MODEL                      │
│                                                                      │
│   Level 1: REACTION                                                  │

│   • Engagement metrics                                               │
│   • Perceived relevance and difficulty                               │

│                                                                      │
│   Level 2: LEARNING                                                  │

│                                                                      │

│   • Consistency of practice                                          │
│   • Transfer to new situations                                       │
│   Method: Follow-up surveys, journal analysis, real-world reports    │
│                                                                      │
│   Level 4: RESULTS                                                   │
│   "What is the broader impact?"                                      │
│   • Spiritual growth indicators                                      │
│   • Community contributions                                          │
│   • Long-term learning sustainability                                │

```

#### 15.5.1 Analysis of a Prophetic Teaching Episode

```
EXAMPLE: Hadith of Jibril (Teaching Iman, Islam, Ihsan)

Teaching Episode Structure:

1. CONTEXT (Attention Capture):
   - Jibril appears as a man to a gathering
   - Unexpected visitor creates curiosity

2. QUESTIONING (Engagement):
   - Jibril asks, "What is Islam?"
   - Prophet responds with detailed answer
   - Jibril confirms: "You have spoken truthfully"

3. PROGRESSION (Structured Sequence):
   - Islam (outward actions) → Iman (inner beliefs) → Ihsan (spiritual excellence)
   - Simple to complex; outward to inward

4. VERIFICATION (Assessment):
   - Witnesses asked to confirm the identity
   - Learners reflect on the extraordinary event

5. CLARIFICATION (Deep Learning):
   - Prophet explains: "That was Jibril, teaching you your religion"

Instructional Design Principles Derived:
  1. Start with an engaging hook (surprise visitor)
  2. Use progressive disclosure (basic → advanced)
  3. Build from concrete to abstract
  4. Include verification of understanding
  5. End with clear synthesis of what was learned
```
